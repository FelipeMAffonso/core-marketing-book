The outside course pattern becomes a chapter microcycle.

The course does not need a separate AI literacy unit after every marketing topic. It needs a repeated work pattern that starts with a marketing decision and ends with a verified artifact.

1. Decision What are we trying to decide? Price, segment, creator, message, crisis response, channel, budget, or research method.
2. Evidence What should AI see? Files, sources, transcripts, survey data, analytics exports, claims, and constraints.
3. Prompt How do we ask? Students write or repair the instruction, rubric, output schema, and examples.
4. Tool Call What can the model use? API request, function call, MCP resource, file search, web search, or app connector.
5. Verification What must a human check? Source quality, missing data, privacy, cost, bias, permissions, and unsupported claims.
6. Artifact What gets saved? Prompt pack, Skill, Custom GPT spec, audit memo, dashboard brief, or agent requirement doc.

These are the kinds of widgets that belong inside the chapters.
