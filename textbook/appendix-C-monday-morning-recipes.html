Monday-Morning Recipe Library

Felipe M. Affonso · Oklahoma State University, Spears School of Business

Every chapter taught a tool. This appendix lets you run them.

A textbook chapter that shows a paste-able prompt is doing more pedagogy than a chapter that names a framework. The chapters in this book ship more than 15 working prompts that turn classroom frameworks into working AI tools. This appendix collects every prompt in one place, with a slash command, the source chapter, the full paste-able body, a walkthrough of how to run it (claude.ai → Projects → upload → paste), instructions for packaging the prompt as an Anthropic Skill, and the per-run cost.

The library is meant to be scanned, not read end to end. Find the recipe you need from the cross-reference table; copy the prompt; run it; come back when you need a different one.

How to use this appendix

Every recipe below is a working system prompt. To run any one of them, open claude.ai (or ChatGPT, Codex, or Claude Code), create a Project (or paste into a system message), drop the prompt into project knowledge or system instructions, attach the input artifact (a campaign image, survey draft, brand brief, dataset), and ask the model to follow the prompt. To turn a recipe into a saved slash command any teammate can run, package the prompt as an Anthropic Skill (folder name + SKILL.md + reference files); see each recipe's packaging block.

CH 0 · SKILL EXAMPLE

/competitor-pestel

PESTEL scan on a competitor brand using web search; returns six-bucket diagnosis with watch-list signals.

---
name: competitor-pestel
description: Run a PESTEL scan on a competitor brand. Use when the
  user asks for a political, economic, social, technological,
  environmental, or legal read on a named brand.
---

# Competitor PESTEL

When invoked with a brand name, do the following:

1. Search the web for news from the last 90 days about the brand,
   the brand's category, and the brand's two largest competitors.
2. Group findings into the six PESTEL buckets. Cite each finding
   with the source URL and publication date.
3. For each bucket, write one sentence naming the marketing
   decision the finding should change.
4. End with three "watch list" signals that, if they move in the
   next quarter, would invalidate the read.

Output format: Markdown with H2 per bucket, bullets per finding,
inline links to sources. Do not invent citations. If a bucket has
no real finding, write "no signal in window" and move on.

How to run. Save as ~/.claude/skills/competitor-pestel/SKILL.md. Restart Claude Code once. Invoke from any chat with /competitor-pestel Stanley. The model loads the Skill, runs the web searches, and produces the structured PESTEL diagnosis.

competitor-pestel/ ├── SKILL.md # the prompt above ├── references/ │ └── pestel-template.md # blank PESTEL framework └── scripts/ └── scan.py # optional fallback when web search rate-limits # install cp -r competitor-pestel ~/.claude/skills/ # invoke /competitor-pestel Stanley

Cost. Free or covered by Claude / ChatGPT subscription (Tier 1). Add about 10-30 cents per run if executed via API with web search and prompt caching on. Source: Chapter 0, §0.5 Skills.

CH 0 · SCHEDULED AGENT

/monday-digest (cron)

Tier-4 example: schedule a recurring agent that runs every Monday at 9 a.m., produces a competitor brief, and posts the summary to a Slack channel.

# ~/agents/monday-digest.sh
#!/usr/bin/env bash
set -euo pipefail
cd "$HOME/agents"

claude --print --dangerously-skip-permissions \
  --output-format text \
  "Run /competitor-pestel Stanley. Then post the result to the
   Slack channel #marketing-research using the slack MCP server.
   Title the message 'Monday Stanley brief — '\$(date +%Y-%m-%d).

# crontab -e, then add this one line:
# 0 9 * * MON /Users/me/agents/monday-digest.sh >> /Users/me/agents/monday.log 2>&1

How to run. Save the script. Make it executable (chmod +x). Add the cron line. The agent runs every Monday at 9 a.m., pulls the PESTEL scan, and posts to Slack. The marketer reviews the brief at 9:30 a.m. with a coffee.

Cost. 1-5 dollars per run depending on model choice and prompt-cache efficiency. The Anthropic API call is the main line item. Source: Chapter 0, §0.7 Tier 4.

03 // chapter 1 — strategy audit

C.3 Chapter 1 — Strategy audit recipe

CH 1 · THREE-DISCIPLINE LENS

/strategy-audit

Score a company on the three Treacy-Wiersema value disciplines, run Levitt's marketing-myopia test, and check the five marketing decisions for evidence-or-absent.

You are reviewing the uploaded materials for a single company.

Score the company on each of the three value disciplines from
Treacy and Wiersema (1995):
  - operational excellence  (cost, reliability, smoothness of the buy)
  - product leadership      (offer itself, design, social meaning)
  - customer intimacy       (long-running relationships, anticipated needs)

For each discipline, give a 0-3 score and quote one literal phrase
from the uploaded materials as evidence. If no evidence is present,
say "not-present" rather than inventing one.

Then run Levitt's marketing-myopia test on the company's own
self-description. Diagnose one of:
  - product-blindness   (defines itself by what it makes)
  - function-only       (defines itself by what its product does)
  - customer-blindness  (defines itself without the buyer's job)

Then check the five marketing decisions:
  1. what to make           2. who to serve
  3. what to charge         4. where to make it available
  5. how to communicate
For each, quote the evidence or mark "not-present."

Finally, flag any place where the declared discipline is
CONTRADICTED by one of the five decisions. That contradiction
is the diagnostic the strategy audit exists to surface.

How to run. Open claude.ai. Click Projects. Create a new project named after the company. Upload three or four recent press releases plus the homepage and About-page copy from the marketing site. Open a chat in the project and paste the prompt above. The model returns a structured audit you can read in a single sitting.

strategy-audit/ ├── SKILL.md # the prompt above └── references/ └── treacy-wiersema-disciplines.md # 1-page summary └── levitt-myopia.md # 1-page summary # install cp -r strategy-audit ~/.claude/skills/ # invoke /strategy-audit

Cost. A few cents per audit on Claude Sonnet with prompt caching on the system message. Failure mode: the model invents biases or evidence the artifact does not support. Recovery prompt: "For every claim, quote the literal phrase from the uploaded materials. If you cannot, mark not-present and move on." Source: Chapter 1, §1.5 Audit.

04 // chapter 3 — bias audit

C.4 Chapter 3 — Bias audit recipe

CH 3 · CIALDINI × KAHNEMAN-TVERSKY

/bias-audit

Audit a campaign artifact (landing page, email, ad script, checkout flow) against Cialdini's seven principles of influence and the Kahneman-Tversky behavioral-economics catalog.

You are a buyer-behavior auditor trained on Cialdini's seven principles of
influence (reciprocity, commitment-consistency, social proof, liking,
authority, scarcity, unity) and on the Kahneman-Tversky behavioral-economics
catalog (loss aversion, mental accounting, endowment effect, anchoring,
availability, representativeness, default effect, System 1 vs System 2).

For the campaign artifact attached, do four things:

1. List every behavioral bias the artifact is currently leveraging. For each,
   quote the exact phrase, image, layout, or interaction that fires it.
2. List every bias the artifact is FAILING to leverage that the situation
   would reward. Name the missed opportunity in one sentence.
3. Flag any bias used in a way the FTC, the buyer, or a journalist would call
   manipulative (false scarcity, fake reviews, dark-pattern defaults).
4. Propose three concrete edits, ordered by expected lift. Each edit names
   the bias, the change, and one sentence of expected effect. No vague advice.

Output format: four numbered sections, plain prose, no preamble, no closer.

How to run. Open claude.ai. Go to Projects. Create a new project called Bias audit. Paste the prompt above into the project's system instructions. Drop in your campaign artifact (screenshot, email body, ad script, checkout flow). Ask the model to audit it. The audit reads as if a senior planner walked through the artifact line by line.

bias-audit/ ├── SKILL.md └── references/ ├── cialdini-7.md # one paragraph per principle └── kahneman-8.md # one paragraph per bias # install cp -r bias-audit ~/.claude/skills/ # invoke /bias-audit ./screenshot.png

Cost. 2-5 cents per audit on Claude Sonnet with prompt caching on the system message. Failure mode: the model invents biases the artifact does not actually leverage. Recovery prompt: "Quote the literal phrase or describe the literal pixel for every bias you name. If you cannot find a literal anchor, mark the bias as not present and move on." Source: Chapter 3, §3.4 Audit.

C.5 Chapter 4 — Marketing research recipes

CH 4 · QUALITATIVE CODING

/open-end-coder

Code 200 to 500 open-ended survey responses into a 6-10 theme scheme with frequency table and representative quotes per theme. Replaces three days of junior-analyst manual coding.

You are coding 247 open-ended survey responses to the question:
"What would make you switch from your current insulated tumbler brand?"

Step 1. Read all 247 responses end-to-end before coding any.
Step 2. Propose a coding scheme: 6 to 10 themes, each with a one-sentence
  definition, mutually exclusive, exhaustive (an "other" theme is OK).
Step 3. Code each response. A response can fit one theme or multiple.
  Flag ambiguous responses for human review.
Step 4. Return the coding scheme, theme assignments, frequency table,
  and the 5 most representative quotes per theme (verbatim, with IDs).

How to run. Open claude.ai. Click Projects. Create a project called Open-end coder. Upload the response file (CSV, Excel, or pasted text). Edit the prompt above to replace the question with yours and the count (247) with your N. Paste the edited prompt into the chat. The model returns the coding scheme, the per-response codes, and the frequency table.

open-end-coder/ ├── SKILL.md └── examples/ └── sample-codings.md # 2-3 worked examples for in-context priming # install cp -r open-end-coder ~/.claude/skills/ # invoke /open-end-coder ./responses.csv

Cost. 30 cents to 1 dollar in API tokens with prompt caching on, for a 25-transcript / 250-response corpus. The same work cost 1,500-3,000 dollars in junior-analyst salary in 2024. Failure mode: the model proposes too few themes (lumps everything into 3 generic categories). Recovery prompt: "Propose at least 6 themes, each grounded in 5 or more verbatim quotes. If a theme has fewer than 5 supporting quotes, dissolve it into an existing theme." Source: Chapter 4, §4.4 Coding.

CH 4 · TIER-1 DESK RESEARCH

tier-1 desk audit

Web-search desk-research audit before commissioning any new study. The cheapest research is the research that prevents an expensive mistake.

Audit publicly available data on women age 25-44 buying 40-ounce
insulated tumblers in the United States, 2018-2020. Cite every source.
Tell me what you cannot find.

How to run. Open claude.ai (or ChatGPT) with web search enabled. Paste the prompt with your category and time range substituted. The model returns a 12-paragraph synthesis with cited URLs. Click every URL to verify. Half will confirm what the team suspected; one or two will surface a dataset the team had not seen; one may contradict an assumption that was about to drive a 200,000-dollar research spend.

/brexi-concept-scorer

Score a product concept against the BREXI rubric (Believable, Relevant, Engaging, X-factor, Inspiring) and return a pass / revise / kill verdict for stage-gate review.

You are a methodologist scoring product concepts against the BREXI rubric.

CONCEPT: [one-paragraph description of the offer, including price tier and channel]
TARGET BUYER: [one line: who, when they buy, what they buy today instead]

Return exactly this structure, nothing else:

BELIEVABLE  (1-5): [score]  -- [one sentence on why]
RELEVANT    (1-5): [score]  -- [one sentence on why]
ENGAGING    (1-5): [score]  -- [one sentence on why]
X-FACTOR    (1-5): [score]  -- [one sentence on why]
INSPIRING   (1-5): [score]  -- [one sentence on why]

WEAKEST DIMENSION: [name the lowest-scoring dimension]
ONE REVISION: [a single concrete change that would lift that score by at least one point]
KILL CRITERION: [pass / revise / kill]

How to run. Open claude.ai. Click Projects. Click New project. Name it BREXI scorer — {your brand}. Drop the parent brand's positioning statement and a one-paragraph buyer profile into project knowledge so the model carries the context across every concept test. Open a chat in the project and paste the prompt, replacing the bracketed lines with the concept under review.

brexi-concept-scorer/ ├── SKILL.md └── references/ └── brand-positioning.md # the brand's positioning statement └── buyer-profile.md # the target buyer profile # install claude skill install ./brexi-concept-scorer # invoke /brexi-concept-scorer

Cost. A typical run lands in under 15 seconds and costs pennies per concept with Anthropic prompt caching on the project knowledge. 08 // chapter 14 — alignment audit

C.8 Chapter 14 — Sales-marketing alignment audit

CH 14 · FOUR-GAP DIAGNOSTIC

/alignment-audit

Audit a B2B firm's sales-marketing alignment for the four standard gap modes (SLA drift, handoff velocity, network centrality, trait-vs-tenure mismatch). Returns scores, named-pattern citations, and a 14-day remediation list.

You are auditing a B2B firm's sales-marketing alignment for the
following four gaps. For each gap, score 0-3 (0 = no evidence,
3 = severe), cite the specific number or pattern in the data
that triggered the score, and propose one concrete fix the head
of revenue can ship in 14 days.

Gap 1. SLA definition drift. Compare the marketing-qualified
       criteria implicit in the lead-score model to the sales-
       qualified criteria implicit in the disposition codes.
       Flag any criterion present in one and absent in the other.

Gap 2. Handoff velocity. From the timestamp data, compute the
       median time from MQL creation to first sales touch. Flag
       any segment where that median exceeds 24 hours, and any
       cohort where the conversion rate drops more than 30 percent
       past the 24-hour mark.

Gap 3. Internal network centrality. From the rep roster, identify
       reps whose deals close without engaging product, legal,
       or customer success in the first 30 days of the cycle.
       Flag isolated reps as a structural risk per Bolander,
       Satornino, Hughes, and Ferris (2015) on relational versus
       positional centrality.

Gap 4. Trait-versus-tenure mismatch. From the cohort retention
       data, compare top-decile first-quarter performers to top-
       decile third-year performers. Flag any cohort where the
       overlap is below 30 percent, per Satornino, Allen, Shi,
       and others on trait-tenure misalignment in B2B sales.

Output: four numbered gap diagnoses; for each, score, evidence,
and one 14-day fix. End with the single most-binding gap to fix
first.

How to run. Open claude.ai (or ChatGPT). Create a new project. Upload (a) last quarter's MQL-to-SQL conversion table from the CRM, (b) an anonymized roster of top-decile and bottom-decile reps with tenure and revenue. Paste the prompt. The model returns a four-gap diagnosis with a numbered remediation list.

Cost. 5-15 cents per audit, with prompt caching on the reference papers. Source: Chapter 14, §14.6 Audit.

CH 18 · MCP TOOL DEFINITION

tool: search_competitors

JSON schema for an MCP tool the model can call. Names the tool, describes when to use it, and types the parameters. The model picks this tool when its task description matches the description field.

{
  "name": "search_competitors",
  "description": "Returns the top 10 brands in a category with weekly review velocity",
  "parameters": {
    "type": "object",
    "properties": {
      "category": { "type": "string", "description": "e.g., insulated drinkware" },
      "lookback_days": { "type": "integer", "default": 7, "minimum": 1, "maximum": 30 }
    },
    "required": ["category"]
  }
}

How to run. Drop this JSON into the model's tool-definitions list when you create the API call. The model will pick the tool when its task matches "find competitors in a category." When the model picks the tool, it returns a JSON call with the parameters; your code runs the actual search; the result returns to the model as a tool-result message.

Cost. Tool definitions are free (just metadata). The model API call is the cost. Source: Chapter 18, §18.2 Tools.

CH 18 · FASTMCP SERVER BOILERPLATE

FastMCP server (Python)

A minimum-viable MCP server in Python that exposes a brand-voice resource and two brand-voice tools (lint, rewrite). Any MCP-speaking client can connect.

from fastmcp import FastMCP

mcp = FastMCP("brand-voice")

@mcp.resource("brandvoice://styleguide")
def styleguide() -> str:
    return open("styleguide.md").read()

@mcp.tool
def lint_brand_voice(text: str) -> dict:
    banned = ["leverage", "synergy", "unlock", "—"]
    hits = [w for w in banned if w in text.lower()]
    return {"compliant": len(hits) == 0, "violations": hits}

@mcp.tool
def rewrite_in_brand_voice(text: str, audience: str) -> str:
    return f"Rewritten for {audience}: {text}"

if __name__ == "__main__":
    mcp.run()

How to run. Save as brand_voice_server.py. Install dependencies with pip install fastmcp. Run with python brand_voice_server.py. Connect from Claude Code, Cursor, or any MCP-speaking client; the server exposes brandvoice://styleguide as a resource and lint_brand_voice and rewrite_in_brand_voice as tools.

Cost. Free to run locally. Hosting a remote MCP server costs the cloud-VM bill (a few dollars per month for low-traffic internal use). Source: Chapter 18, §18.2 Tools.

C.10 Chapter 6 — Perceptual map recipe

CH 6 · POSITIONING-TO-MAP

/perceptual-map-from-statements

Take ten brands' positioning statements plus two attribute axes the target buyer weighs, return a coral 2D scatter PNG with each brand placed and the empty quadrant the brand manager should consider next. The artifact a category review needs at 9 a.m.

~/.claude/skills/perceptual-map-from-statements/
├── SKILL.md           # frontmatter + instructions
├── references/
│   └── coral-palette.md  # locked hex codes (#DA7756, #B8553A, #1a1a1a)
└── scripts/
    └── render.py          # matplotlib renderer

# SKILL.md
---
name: perceptual-map-from-statements
description: Plot ten brands on a 2D perceptual map from their positioning statements and two attribute axes.
arguments:
  - name: brands_yaml
    description: Path to YAML with ten brand positioning statements.
  - name: x_axis
    description: Pair of opposing attributes for the x-axis (e.g. "premium ↔ value").
  - name: y_axis
    description: Pair of opposing attributes for the y-axis (e.g. "traditional ↔ modern").
---

For every brand in {{brands_yaml}}, score the positioning statement on
{{x_axis}} from -1 to +1 and on {{y_axis}} from -1 to +1, justifying each
score in one clause that quotes a literal phrase from the statement. Save
the scored table to data/scores.csv. Then call scripts/render.py with the
csv and the two axis labels; verify perceptual-map.png exists; report the
empty quadrant the brand manager should consider next.
# scripts/render.py
import csv, sys, matplotlib.pyplot as plt
CORAL, DEEP, INK = '#DA7756', '#B8553A', '#1a1a1a'
xs, ys, names = [], [], []
with open(sys.argv[1]) as f:
    for row in csv.DictReader(f):
        xs.append(float(row['x'])); ys.append(float(row['y']))
        names.append(row['brand'])
fig, ax = plt.subplots(figsize=(7.2, 4.8), dpi=200)
ax.axhline(0, color='#888', lw=0.5, ls='--'); ax.axvline(0, color='#888', lw=0.5, ls='--')
ax.scatter(xs, ys, s=160, c=CORAL, edgecolor=DEEP, linewidth=1.4, zorder=3)
for x, y, n in zip(xs, ys, names):
    ax.annotate(n, (x, y), xytext=(0, 12), textcoords='offset points',
                ha='center', color=INK, fontsize=10, fontweight='600')
ax.set_xlim(-1.1, 1.1); ax.set_ylim(-1.1, 1.1)
ax.set_xlabel(sys.argv[2], color=INK); ax.set_ylabel(sys.argv[3], color=INK)
ax.spines[['top','right']].set_visible(False)
plt.tight_layout(); plt.savefig('perceptual-map.png', dpi=200, facecolor='white')

How to run. Drop the folder above into ~/.claude/skills/. Build a brands.yaml with the ten positioning statements (the firm's plus nine competitors, copied verbatim from each brand's website or last earnings deck). Open Claude Code in any project directory and invoke /perceptual-map-from-statements brands.yaml "premium ↔ value" "traditional ↔ modern". The Skill scores each statement on both axes with a literal-phrase justification, writes data/scores.csv, runs the renderer, and prints the empty-quadrant diagnosis. The PNG drops next to the input file in 25 seconds, ready for the deck.

Cost. About 4 cents in API tokens with prompt caching on. Total runtime, about 25 seconds. Failure mode: axes that are not genuinely opposing (premium vs premium-er) flatten the map into a line because every brand scores positive. Recovery prompt: "If both axis ends measure the same direction of the same attribute, the layout will degenerate. Swap one end for its semantic opposite (e.g. premium ↔ value) and re-run." Source: Chapter 6, §6.6 Perceptual maps.

C.11 How recipes compose

The 18 recipes above span the four 2026 AI-workflow tiers. Tier 1 (chat-search) is the desk audit. Tier 2 (Skills) is the bulk of the library: /strategy-audit, /bias-audit, /competitor-pestel, /perceptual-map-from-statements, /brexi-concept-scorer, /stepps-scorer, /alignment-audit, /open-end-coder. Tier 3 (MCP servers) is the config, the tool definition, and the FastMCP boilerplate. Tier 4 (Codex / Claude Code on a schedule) is the cron example.

The recipes compose in two patterns. The first pattern is the upgrade ladder: a marketer running a recipe weekly as a chat prompt eventually packages it as a Skill, eventually wires the data source through MCP, eventually schedules the workflow on cron. Each upgrade reduces friction; each upgrade reduces the marketer's per-week time investment from minutes to seconds to zero. The second pattern is the chain: /competitor-pestel → /open-end-coder on the open-end question "what are competitors doing better" → /brexi-concept-scorer on a concept that addresses the gap. The chain is the structural answer to the capstone chapter's central question: a five-component agent is built by chaining recipes that each handle one component.

The chapters teach the principle behind each recipe; this appendix lets the principle be acted on. The canonical readings appendix backs each recipe to its theoretical source; the extended cases appendix shows the recipes operating against real 2024-2026 brand decisions. The three appendices together hold the depth a chapter cannot afford in body prose.
